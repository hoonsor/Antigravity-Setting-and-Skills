# AntiGravity 異機還原指南

當您在新的電腦上克隆 (Clone) 或還原了此設定檔倉庫後，MCP 工具（如 GitHub、Google Drive）會因為缺少包含金鑰的 `mcp_config.json` 而無法載入。請跟隨以下步驟還原：

## 準備工作
在開始前，您需要準備好以下三項機密資訊（為求安全，這些並未備份在 Git 中）：
1. **GitHub Personal Access Token** (以 `ghp_` 開頭)
2. **gcp-oauth.keys.json** 內容 (GCP 專案的 OAuth 客戶端 ID)
3. **.gdrive-server-credentials.json** 內容 (GCP 的服務帳戶或是 Oauth 授權憑證)

## 還原步驟 (一鍵 Prompt)

請直接複製以下 Prompt，並貼給新電腦上的 AntiGravity 即可，它會自動引導您完成後續設定：

```text
【異機環境還原任務】
請幫我執行以下 MCP 工具還原流程：
1. 讀取當前電腦使用者家目錄下 `.gemini\config\mcp_config.json.template` 的內容。
2. 在同目錄下自動產生出 `mcp_config.json` 檔案。
3. 詢問並引導我提供以下三項機密資訊：
   - GitHub Personal Access Token
   - gcp-oauth.keys.json 的內容
   - .gdrive-server-credentials.json 的內容
4. 當我提供後，請幫我把 Google 的兩個憑證檔建立在當前電腦使用者家目錄下 `.gemini\` 目錄下，並自動將這三個金鑰正確替換到 `mcp_config.json` 檔案中。
5. 完成後，請提示我重啟 AntiGravity 以載入 MCP 工具，並執行 `00-install-all` 技能以確保所有懶人包皆已正確安裝。
```

## 注意事項
- 重啟 AntiGravity 後，第一次連線時底層工具 (`npx` 與 `docker`) 會自動在背景下載並啟動對應的 MCP 工具，**完全無需手動打指令安裝**。
- **請絕對不要將填寫好金鑰的 `mcp_config.json` 或 Google 憑證檔 commit 到 Git 備份中！**
